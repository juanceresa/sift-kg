## 3.1 配置管理模块 `config.py`

**职责**: 管理应用程序配置，支持多层次配置优先级。

### 设计思路

配置优先级从高到低:
1. CLI 标志（最高优先级）
2. 环境变量 (`SIFT_` 前缀)
3. `.env` 文件
4. `sift.yaml` 项目配置文件
5. 默认值（最低优先级）

这种设计允许:
- CLI临时覆盖配置
- 环境变量部署敏感信息（API密钥）
- `.env` 本地开发
- `sift.yaml` 项目默认配置

### 核心类 `SiftConfig`

```python
class SiftConfig(BaseSettings):
    # LLM Configuration
    llm_model: str
    llm_max_concurrent: int = 4
    llm_max_retries: int = 3
    
    # API Keys
    openai_api_key: str | None = None
    anthropic_api_key: str | None = None
    google_api_key: str | None = None
    mistral_api_key: str | None = None
    
    # OCR Configuration
    ocr_enabled: bool = False
    ocr_backend: OcrBackend = OcrBackend.tesseract
    ocr_language: str = "eng"
    
    # Domain Configuration
    domain: str | None = None
    domain_path: str | None = None
    
    # Output Configuration
    output_dir: Path = Path("output")
    
    # Chunking
    chunk_size: int = 10_000
```

使用 `pydantic-settings` 实现类型安全的配置验证。

**关键设计点**:
- 自定义 `_ProjectYamlSource` 类从 `sift.yaml` 读取配置
- 在验证后自动将API密钥导出到环境变量供LiteLLM使用
- 自动验证：Ollama本地模型不需要API密钥，其他提供商必须提供
- 自动创建输出目录，如果不存在

### 配置示例

`sift.yaml`:

```yaml
llm_model: "gpt-4o-mini"
llm_max_concurrent: 4
domain: "osint"
output_dir: "./my-output"
ocr_enabled: false
```

`.env`:

```
OPENAI_API_KEY=sk-...
ANTHROPIC_API_KEY=sk-...
```

环境变量覆盖:

```bash
export SIFT_LLM_MODEL=gpt-4o
sift extract ./docs
```

## 3.2 文档摄取模块 `ingest/`

**职责**: 从各种格式的文档中提取纯文本，支持OCR处理扫描文档。

### 组件结构

```
ingest/
├── base.py                    # 协议定义
├── reader.py                  # 编排
├── chunker.py                 # 分块
├── ocr.py                     # OCR抽象
├── kreuzberg_extractor.py     # 默认后端 (75+格式)
└── pdfplumber_extractor.py    # 遗留PDF后端
```

### 协议设计 `base.py`

定义 `TextExtractor` 协议，遵循协议模式：

```python
class TextExtractor(Protocol):
    def extract_text(self, file_path: Path) -> str: ...
```

任何实现 `extract_text` 方法的类都可以作为提取后端，易于扩展。

### 文档读取编排 `reader.py`

主要职责:
1. 根据配置选择正确的提取后端
2. 对PDF自动检测是否需要OCR：
   - 如果提取得到非空文本 → 直接使用
   - 如果提取得到空文本 → 回退到OCR
3. 支持混合文件夹（既有文本PDF又有扫描PDF），对用户透明

核心函数: `read_document(file_path: Path, config) → str`

### 文本分块 `chunker.py`

将长文本分割成LLM可处理的块，默认10,000字符。

设计特点:
- 按段落边界分割，避免在句子中间切断
- 尽量保持块接近但不超过目标大小
- 保留每个块的原始位置信息
- 对于短文档（小于chunk_size）整个作为一个块

### OCR抽象 `ocr.py`

支持多个OCR后端，统一接口：

| 后端 | 特点 | 需要API |
|------|------|--------|
| `tesseract` | 默认，本地，开源 | 否 |
| `easyocr` | 更好精度，更大内存 | 否 |
| `paddleocr` | 中文支持更好 | 否 |
| `gcv` | Google Cloud Vision，云端高精度 | 是 |

### 提取后端

1. **Kreuzberg** (默认):
   - 支持75+文件格式：PDF, DOCX, XLSX, PPTX, HTML, EPUB, JPG, PNG, TIFF等
   - 基于`textract`但更现代化
   - 维护活跃，依赖问题少

2. **PDFPlumber** (legacy):
   - 仅支持PDF/DOCX/TXT/HTML
   - 保留用于向后兼容
   - 对某些旧PDF格式提取更好

### 设计特点

- **可插拔后端**: 协议模式易于添加新后端
- **自动OCR回退**: 用户不需要手动判断哪些文档需要OCR
- **本地处理**: 所有文本处理都在本地完成，文档不会离开你的机器
- **混合支持**: 同一项目可以混合文本文档和扫描文档
