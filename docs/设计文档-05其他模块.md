### 3.7 叙事生成 `narrate/`

**职责**: 使用LLM生成 prose 叙事报告，包括社区标签和实体描述。

**组件**:
- `generator.py`: LLM调用编排
- `prompts.py`: 提示词模板

**输出**:
- `output/narrative.md`: 完整叙事报告
  - 概览: 实体、关系、社区统计
  - 关键关系链: 最高度实体之间的关系
  - 时间线（如果有日期信息）
  - 按社区分组的实体描述
- `output/entity_descriptions.json`: 实体描述缓存供可视化使用
- `output/communities.json`: 社区标签

**生成流程**:
1. 如果没有社区标签，先生成每个社区的描述性标签
2. 对每个实体生成描述，基于上下文证据
3. 生成整体叙事，结构化为可读报告
4. 实体描述用主动语态，聚焦具体行动，不是角色摘要

**选项**:
- `--communities-only`: 只重新生成社区标签，成本很低（~$0.01）

### 3.8 可视化 `visualize.py`

**职责**: 生成交互式浏览器可视化，基于pyvis force-directed布局。

**功能特性**:

- **社区区域**: 彩色凸包分组，一眼看到图结构
- **悬停预览**: 悬停节点显示名称和连接
- **焦点模式**: 双击节点隔离邻域，箭头键逐步浏览
- **踪迹面包屑**: 记录探索路径，边保持高亮
- **搜索**: 在图中搜索实体
- **切换开关**: 按类型、社区、关系类型切换显示
- **来源文档过滤**: 只显示来自特定文档的实体
- **度过滤**: 隐藏低度节点

**预过滤选项** (CLI参数):
- `--neighborhood "entity" --depth N`: 只显示实体的N跳邻域
- `--top K`: 只显示Top K中心节点及其邻居
- `--community "Community X"`: 只显示特定社区
- `--source-doc doc_id`: 只显示来自特定文档的实体
- `--min-confidence 0.8`: 隐藏低置信度节点/边

**输出**: 生成 `output/graph.html`，在浏览器自动打开。

**技术实现**:
- 使用pyvis生成HTML/JavaScript
- 力导向布局自动聚类
- 社区着色基于Louvain检测结果

### 3.9 导出 `export.py`

**职责**: 导出知识图谱到多种标准格式，供外部工具分析。

**支持格式**:

| 格式 | 用途 | 输出文件 |
|------|------|----------|
| `graphml` | Gephi, yEd, Cytoscape | `graph.graphml` |
| `gexf` | Gephi原生格式 | `graph.gexf` |
| `sqlite` | SQL查询, Datasette, DuckDB | `graph.sqlite` (nodes + relations表) |
| `csv` | 通用分析 | `csv/entities.csv`, `csv/relations.csv` |
| `json` | 原生格式 | `graph.json` |

使用NetworkX内置导出功能，保持所有属性。

### 3.10 CLI 接口 `cli.py`

**职责**: 命令行界面，使用Typer构建。

**命令层次**:

```
sift init                    # 初始化项目，创建sift.yaml + .env.example
sift domains                 # 列出所有可用域
sift extract <directory>     # 抽取实体和关系
sift build                   # 构建知识图谱
sift resolve                 # LLM提出合并建议
sift review                  # 交互式审核合并建议
sift apply-merges            # 应用已批准的合并
sift narrate                 # 生成叙事报告
sift view                    # 在浏览器中交互式查看
sift search "query"          # CLI搜索实体
sift query "entity"          # 查询实体邻域
sift topology                # 输出拓扑结构JSON（用于AI agent）
sift info                    # 输出项目统计信息
sift export <format>         # 导出到指定格式
```

**特性**:
- 丰富的帮助文本
- 命令补全
- 富文本终端输出
- 进度条显示抽取进度
- 成本跟踪显示当前花费

### 3.11 库 API `pipeline.py`

**职责**: 提供可编程Python API，供Jupyter笔记本、脚本、web应用使用。

**导出函数**:
- `load_domain()`: 加载域配置
- `run_extract()`: 运行抽取
- `run_build()`: 运行构建，返回KnowledgeGraph对象
- `run_resolve()`: 运行实体解析
- `run_narrate()`: 运行叙事生成
- `run_export()`: 运行导出
- `run_view()`: 运行可视化
- `run_pipeline()`: 运行完整管道（extract → build → narrate）

所有参数和CLI对应，支持相同配置选项。
