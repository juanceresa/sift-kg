### 3.5 知识图谱核心 `graph/`

**职责**: 知识图谱数据结构、构建和存储。

**核心类 `KnowledgeGraph` (`knowledge_graph.py`)**:
- 封装 NetworkX `MultiDiGraph`（支持多边有向图）
- 每个实体是节点，每个关系是边
- 节点属性: `id`, `entity_type`, `name`, `confidence`, `source_documents`, `attributes`
- 边属性: `source`, `target`, `relation_type`, `confidence`, `evidence`, `source_document`, `mentions`, `support_count`

**主要方法**:
- `add_entity()`: 添加或更新实体，如果已存在则合并（更新置信度，扩展来源文档）
- `add_relation()`: 添加关系，如果同一(source, relation_type, target)已存在则聚合置信度
- `get_entity()`: 按ID获取实体
- `get_relations()`: 获取实体的所有关系（支持in/out/both方向）
- `export()`: 导出为JSON可序列化字典
- `save()`: 保存到JSON文件
- `load()`: 从JSON文件加载

**置信度聚合策略**:
- `product_complement` (默认): 1 - Π(1 - cᵢ)，独立弱信号互相增强
- `mean`: 简单平均值
- `max`: 最大置信度

**规范键设计**:
- 同一(source, relation_type, target)三元组共享一个规范键 `source|relation_type|target`
- 多次提及自动聚合到同一条边，累积支持计数和来源文档
- 置信度自动聚合，保持最新最佳证据的evidence和source_document

**组件**:

1. **`builder.py` - 图谱构建**
   - 从所有文档抽取结果构建图谱
   - 在添加节点前运行预去重
   - 修复LLM可能颠倒的边方向（根据域定义的source_types/target_types检查）
   - 标记低置信度关系供后续审核
   - 调用社区检测

2. **`prededup.py` - 预去重**
   - 第一层确定性去重，不需要LLM，不花钱
   - Unicode标准化
   - 去除常见头衔前缀（Dr., Mr., Judge等35+种）
   - 名词单复数归一化
   - SemHash模糊匹配（阈值0.95）捕捉近似拼写
   - 基于KGGen论文技术

3. **`postprocessor.py` - 后处理**
   - 修复反向边方向
   - 移除冗余边
   - 标记低置信度关系

4. **`communities.py` - 社区检测**
   - 使用Louvain算法检测社区
   - 每个实体分配到一个社区
   - 用于可视化着色和叙事分组

**存储格式**:
输出JSON格式包含:
- `metadata`: 创建时间、更新时间、实体数、关系数、类型统计
- `nodes`: 所有节点及其属性
- `links`: 所有边及其属性

### 3.6 实体消歧 `resolve/`

**职责**: 检测和解决实体重复，人机协作保证准确性。

**三层架构**:
1. **预去重** (构建时自动) - 处理明显重复
2. **LLM建议** - LLM批量比较提出合并建议
3. **人工审核** - 用户批准/拒绝，不自动合并

**组件**:

1. **`models.py` - 数据模型**
   - `MergeProposal`: 合并建议，包含规范ID、成员列表、状态(DRAFT/CONFIRMED/REJECTED)、原因
   - `RelationReview`: 需要审核的关系，状态管理

2. **`resolver.py` - LLM建议生成**
   - 默认: 按字母顺序分批，重叠批处理保证重复不会漏掉
   - 可选: 嵌入聚类（需要 `sentence-transformers`），语义相似名称聚在一起比较，解决拼写差异大但语义相同的情况
   - 对每个批次，LLM找出指代同一现实世界事物的实体
   - 输出到 `merge_proposals.yaml`，所有建议初始状态DRAFT

3. **`clustering.py` - 嵌入聚类**
   - 使用all-MiniLM-L6-v2生成实体名称嵌入
   - KMeans聚类，语义相似聚到同一批
   - 解决字母顺序分隔开的"Robert Smith"和"Bob Smith"问题

4. **`engine.py` - 图手术引擎**
   - 应用已确认的合并到图谱
   - 步骤:
     1. 将成员节点数据合并到规范节点（保留规范值，扩展来源文档）
     2. 重写所有指向/来自成员的边到规范节点
     3. 移除成员节点
     4. 移除合并产生的自环
   - 统计: 返回应用的合并数、移除的节点数、移除的自环数

5. **`reviewer.py` - 交互式审核**
   - 终端交互式UI，逐个审核DRAFT建议
   - 高置信度自动批准（默认>0.85）
   - 低置信度自动拒绝（默认<=0.5）
   - 支持用户调整阈值
   - 支持跳过、批准、拒绝操作

6. **`io.py` - I/O操作**
   - 读写merge_proposals.yaml和relation_review.yaml
   - 保留历史决策，多次运行增量处理

**设计原则**:
- **从不自动合并**: LLM只提建议，用户做最后决定
- **增量**: 之前批准/拒绝的决定保留，只添加新建议
- **可追溯**: 每个合并保留原因和LLM置信度
- **两种审核方式**: 交互式终端适合快速审核，直接编辑YAML适合高精度需求场景
